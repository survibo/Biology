아래 형식에 맞춰 관련 시험 문제 약 50개 정도 JSON으로 만들어줘.
자신이 문제를 보고 정말 '시험에 나올만 한지' 파악해서 문제로 만들어

규칙:
1. type은 반드시 "single" / "multi" / "any" 중 하나
   - single: 정답 표현이 하나로 고정된 경우 (answers[0] 하나만 정답)
   - multi: 빈칸이 여러 개이며 answers 배열이 빈칸 순서와 대응됨, 모두 맞아야 정답
   - any: 빈칸은 하나지만 인정되는 표현이 여럿인 경우 (예: "심근경색" / "MI" / "심근경색증"), answers 배열에 모두 나열하며 그 중 하나만 입력해도 정답 처리
   - 단, 정규화에 의해 띄어쓰기, 하이픈, 일부 문장부호, 영문 대소문자는 무시되므로 사실상 같은 표현은 any로 처리하지 않아도 됨

2. multi가 시험 대비에 더 유리한 경우 적극적으로 사용할 것
3. 답이 문제 안에 노출되지 않도록 할 것
4. 주석 없이 순수 JS export 코드만 출력할 것
5. 코드 블럭으로 출력할 것
6. 중요 / 생성 후 다시 한번 스스로 검토하여 아래 항목 수정:
   - 답이 문제에 노출된 경우
   - type 또는 answers 배열 개수가 잘못 지정된 경우
   - answers 누락 또는 오류

export const questions = [
  {
    id: 1,
    section: "섹션명",
    prompt: "예시 문제에서 ____ 이다.",
    answers: ["정답"],
    type: "single",
  },
  ...
];
